**Replace the placeholder src/commands/env.rs** with a full implementation:

1. Import `anyhow::Result`.
2. Create `pub fn execute() -> Result<()>` that:
   - Calls `super::run::resolve_profile(None)?` to get `(name, dir)`. Uses None because `clmux env` always uses the active profile (no --profile flag per D-08).
   - Iterates `std::env::vars()` and for any key starting with `"CLAUDE_"` or `"ANTHROPIC_"`, prints `unset {key};` to stdout. This ensures `eval "$(clmux env)"` strips leaked vars from the parent shell (matches what `clmux run` does per D-01).
   - Prints `export CLAUDE_CONFIG_DIR="{dir}";` where `{dir}` is `dir.display()` (per D-07). Values MUST be double-quoted for shell safety.
   - Prints `export CLMUX_PROFILE="{name}";` (per D-09).
   - Returns `Ok(())`.

Every output line must be either `unset ...;` or `export ...=...;` -- valid POSIX shell syntax.

**Enhance src/commands/status.rs:**

After the existing three println! lines (Profile, Path, Status), add:

1. A "Config:" line showing what CLAUDE_CONFIG_DIR would be set to:
   ```rust
   println!("Config:  CLAUDE_CONFIG_DIR={}", dir.display());
   ```

2. If the directory exists, show the number of items (files and subdirectories) inside it. This is a rough health indicator that respects DATA-04 (opaque directories -- we count items but never parse them):
   ```rust
   if dir_exists {
       let item_count = std::fs::read_dir(&dir)
           .map(|entries| entries.count())
           .unwrap_or(0);
       println!("Items:   {} file(s)", item_count);
   }
   ```

This satisfies LNCH-06's "directory health" requirement beyond just exists/missing.
