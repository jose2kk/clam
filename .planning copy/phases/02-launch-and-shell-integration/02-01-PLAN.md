  <action>
Create src/commands/run.rs with the following implementation:

1. A `pub(crate) fn resolve_profile(profile_override: Option<&str>) -> anyhow::Result<(String, std::path::PathBuf)>` helper that:
   - If `profile_override` is Some: calls `crate::profile::validate_profile_name(name)`, then `crate::config::load()` and checks the profile exists in `cfg.profiles`, bails with "Profile '{name}' not found. Run `clmux list` to see available profiles." if not found. Does NOT modify state.toml (per D-05).
   - If `profile_override` is None: calls `crate::state::load()` and extracts `st.active`, bails with "No active profile. Run `clmux add <name>` to create one." if None.
   - Resolves directory via `crate::paths::profile_dir(&name)`.
   - Returns `(name, dir_path)`.

2. A `pub fn execute(profile_override: Option<&str>, args: &[String]) -> anyhow::Result<()>` function that:
   - Calls `resolve_profile(profile_override)` to get (name, dir).
   - Checks `dir.is_dir()` and bails with "Profile directory for '{name}' is missing at {path}. Run `clmux add {name}` to recreate it." if not a directory (Pitfall 4 from research).
   - Determines binary and cmd_args: if `args.is_empty()`, binary = "claude", cmd_args = empty (per D-03). Otherwise binary = `args[0]`, cmd_args = `args[1..]` (per D-04).
   - Creates `std::process::Command::new(&binary)` with `.args(&cmd_args)`.
   - Iterates `std::env::vars()` and calls `cmd.env_remove(&key)` for any key starting with `"CLAUDE_"` or `"ANTHROPIC_"` (per D-01). Does NOT strip `CLMUX_*` vars.
   - Calls `cmd.env("CLAUDE_CONFIG_DIR", &dir)` and `cmd.env("CLMUX_PROFILE", &name)` (per D-02).
   - Uses `std::os::unix::process::CommandExt` trait and calls `let err = cmd.exec();` (per D-06).
   - Since exec() only returns on error, returns `Err(anyhow::anyhow!("Failed to execute '{}': {}. Is it installed and in your PATH?", binary, err))`.

Do NOT use `env_clear()` (would strip PATH, HOME, etc.).
Do NOT use `Command::spawn()` + `wait()` (per D-06, use exec()).
Do NOT add the `which` crate (execvp handles PATH lookup).
